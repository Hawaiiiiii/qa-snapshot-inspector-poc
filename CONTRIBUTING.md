# Contributing

Thanks for your interest in improving QUANTUM Inspector.

## Quick start

1. Fork the repo and create a branch from `develop`.
2. Make your changes with focused commits.
3. Ensure the app launches locally.
4. Open a pull request targeting `develop`.

## Branching (GitFlow)

- `main`: stable releases
- `develop`: active integration
- `feature/*`: new work
- `hotfix/*`: urgent fixes
- `release/*`: pre-release stabilization

## Commit style

Use concise, imperative commit messages.

Examples:
- "fix: handle invalid XML dumps"
- "docs: update README"

## Code style

- Keep changes minimal and focused.
- Preserve existing UI patterns and naming.
- Avoid unrelated formatting changes.

## Reporting issues

Please include:

- OS and Python version
- Steps to reproduce
- Sample snapshot or dump if available
- Screenshots (optional)
